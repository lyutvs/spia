# Manual integration check

The automated `frontendTypecheck` Gradle task verifies that the generated SDK
compiles under `tsc --strict`. Runtime behavior against a real Spring server
is a separate manual step — follow this procedure before each release to
confirm the SDK actually talks to the backend.

## Prerequisites

- JDK 21 on `JAVA_HOME`
- Node.js 18+ and npm
- `npm install` has been run in `app/frontend` (the Gradle build does this
  automatically, so running `./gradlew :app:build` once is enough).

## Procedure

1. **Terminal A** — start the Spring server:
   ```bash
   JAVA_HOME=... ./gradlew :app:bootRun
   ```
2. **Terminal B** — run the integration script:
   ```bash
   cd app/frontend && npm run integration
   ```

The script exits 0 when every endpoint responds without an exception and 1
otherwise. The SDK is regenerated by every `:app:build`, so make sure the
server you're hitting was built from the same commit as the SDK.

## What it exercises

- `GET /api/users/{id}` — path variable, nested DTO in response
- `POST /api/users` — request body
- `PUT /api/users/{id}` — path variable + partial body
- `GET /api/users/list` — optional `@RequestParam` pagination
- `GET /api/users/{id}/wrapped` — multi-parameter generic response
- `DELETE /api/users/{id}` — void response
